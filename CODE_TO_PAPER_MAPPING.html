VOID: Code-to-Paper Mapping

A structured mapping between the paper “VOID: Video Object and Interaction Deletion” and its implementation in the public repository. This page highlights the core logic path, exact source references, and the most important reproducibility-sensitive engineering details.

RepositoryNetflix/void-model
FocusInteraction-aware masking, inpainting, refinement

Executive Summary

The repository implements the paper’s core idea through a staged pipeline: SAM2 segmentation, VLM reasoning over affected objects, quadmask construction, quadmask-conditioned CogVideoX inpainting, and an optional warped-noise refinement pass.

  • Direct paper match: affected-object reasoning is explicitly encoded in the mask-generation pipeline.
  • Important heuristic: the first frame forcibly clears grey-valued affected regions before final quadmask export.
  • Reproducibility concern: config advertises num_inference_steps = 50, while pass-1 inference uses 30 directly in code.

Critical Audit Notes

  • Prompting mismatch risk: the model uses a clean-background description rather than a direct removal command.
  • Mask-value sensitivity: repeated re-quantization suggests the method is sensitive to interpolation and codec artifacts.
  • Hidden heuristic: the first-frame grey suppression is practical but may materially affect behavior.
  • Checkpoint adaptation: mask-conditioned transformer channels require special loading logic instead of naive checkpoint restoration.
