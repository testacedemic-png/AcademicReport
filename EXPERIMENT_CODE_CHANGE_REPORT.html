Per-Experiment Code Change Plan

1. Ablate the affected-region mask

Experiment: compare primary-object-only mask, full quadmask, and corrupted/imperfect affected-region masks.

Files and likely line ranges:

  • VLM-MASK-REASONER/stage4_combine_masks.py — 20–53, 101–115
  • videox_fun/utils/utils.py — 374–413
  • config/quadmask_cogvideox.py — 31–49
  • new helper script under scripts/ or inference/

What to change: add mask-generation modes, optionally suppress 127 affected regions, and add corruption transforms such as dilation, erosion, temporal dropout, and jitter.

Minimal path: post-process quadmask_0.mp4 into variant masks before inference.

Effort: Low–Medium

2. Remove the VLM reasoning stage

Experiment: replace Gemini-based affected-object detection with no affected objects, heuristic motion masks, or simple bbox expansion.

Files and likely line ranges:

  • VLM-MASK-REASONER/stage2_vlm_analysis.py — 256–376, 665–740
  • VLM-MASK-REASONER/stage3a_generate_grey_masks_v2.py — around 338–383
  • VLM-MASK-REASONER/run_pipeline.sh — around 44–66

What to change: add alternative stage-2 backends and emit synthetic vlm_analysis.json files for heuristic modes.

Minimal path: bypass stage-2 and emit empty affected_objects.

Effort: Medium

3. Pass-1 vs Pass-2 study

Experiment: compare pass 1 only vs pass 1 + pass 2.

Files and likely line ranges:

  • inference/cogvideox_fun/predict_v2v.py — 131–184
  • inference/cogvideox_fun/inference_with_pass1_warped_noise.py — 1–16, 42–136, 219–239
  • inference/pass_2_refine.sh — around 25–67

What to change: mostly orchestration only; add a wrapper to run pass 1, optionally run pass 2, and standardize outputs.

Minimal path: create scripts/run_ablation_pass1_vs_pass2.sh.

Effort: Low

4. Denoising-step discrepancy study

Experiment: test 20 / 30 / 50 / 75 denoising steps.

Files and likely line ranges:

  • inference/cogvideox_fun/predict_v2v.py — 172–179
  • config/quadmask_cogvideox.py — 40–47

What to change: replace hardcoded 30 with config.video_model.num_inference_steps.

Minimal path: single-line fix + sweep runner.

Effort: Low

5. Prompt sensitivity study

Experiment: vary minimal, detailed, wrong, and empty prompts.

Files and likely line ranges:

  • videox_fun/utils/utils.py — 335–337
  • input data folders containing prompt.json
  • optional experiment runner script

What to change: no model code change needed if editing prompt.json; optionally add prompt override CLI support near inference arg parsing and get_video_mask_input(...) call sites.

Minimal path: duplicate input folders with different prompt variants.

Effort: Low

6. Mask corruption robustness

Experiment: inject under-mask, over-mask, temporal jitter, and first-frame corruption.

Files and likely line ranges:

  • VLM-MASK-REASONER/stage4_combine_masks.py — 20–53, 101–115
  • videox_fun/utils/utils.py — 374–413
  • new utility script under scripts/

What to change: add perturbation functions for mask videos and save perturbed variants as new quadmask files.

Minimal path: separate offline mask-corruption tool.

Effort: Medium

Files and likely line ranges:

Minimal path: evaluation-layer change only.

Summary

Most of the best experiments for VOID do not require risky architecture rewrites. The highest-value path is to expose hidden heuristics, remove hardcoded inference choices, add reproducible mask ablation tooling, and separate pass-1 from pass-2 contributions.
